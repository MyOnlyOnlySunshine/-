字符编码

* 字符集：很多字符的集合，如文字，标点，图形，数字等的集合；常见的有：ASCII字符集、ISO 8859字符集、GB2312字符集、BIG5字符集、GB18030字符集、Unicode字符集等。
*字符编码：是以二进制的数字来对应字符集的字符，计算机需要对各种字符进行编码，才能对各种文字进行处理。
*编码：每个字符用单字节或多字节存储，用哪些字符存储，字符集与编码一般同时制定。

常用编码

* ASCII（American Standard Code for Information Interchange，美国信息互换标准代码）：单字节编码，基于拉丁字母的一套电脑编码系统，主要用于显示现代英语，而其扩展版本EASCII则可以勉强显示其他西欧语言，只能显示26个基本拉丁字母、阿拉伯数目字和英式标点符号。
*ANSI编码(American National Standards Institute，美国国家标准学会)：多字节编码，系统里的英文和符号就使用单字节的 ASCII（0x00~0x7f），而对于汉字之类的本地化字符编码，就采用 0x80~0xFF 范围内的多个字节来表示，不同的国家和地区制定了不同的标准，由此产生了GB2312,BIG5,JIS等各自的编码标准，在非 Unicode 环境下，由于不同国家和地区采用的字符集不一致，很可能出现乱码问题。
*Unicode（Unicode Character Set）：国际组织制定的 UNICODE 无国界字符集，各种语言中的每一个字符设定了统一并且唯一的数字编号，便于国际交流，这种编码方式改为存放每个字节在unicode字符集中的序号。
  • ucs-2:两个字节编码（常用） ucs-4: 四个字节编码

各大编码区别

*GB2312（1980年发布,GB-"国标"汉语拼音首字母）:ASCII码中小于127的字符的意义与原来相同，但两个大于127的字符连在一起时，就表示一个汉字，高字节在前，低字节在后；共收录6763个汉字，其中一级汉字3755个，二级汉字3008个；同时收录了包括拉丁字母、希腊字母、日文平假名及片假名字母、俄语西里尔字母在内的682个全角字符。
  • 全角字符 ：对ASCII 里本来就有的数字、标点、字母都统统重新编了两个字节长的编码；
  • 半角字符 ：之前127号以下的那些就叫"半角"字符；
*GBK（1995年发布，“国标扩展”）：由于中国汉字太多，6763个汉字无法满足要求，故不要求第二个字符一定大于127， 只要第一个字符大于127就默是汉字，它是GB2312的扩展，收录了21003个汉字，支持国际标准ISO/IEC10646-1和国家标准GB13000-1中的全部中日韩汉字，并包含了BIG5编码中的所有汉字。
*GB18030：分为 2000与2005两个版本是GBK的扩展，是目前我国计算机系统必须遵GB18030-2000循的基础标准之一；
  • GB18030-2000 ：GBK的取代版本，作为国家标准强制执行
  • GB18030-2005：是我国自主研制的以汉字为主并包含多种我国少数民族文字（如藏、蒙古、傣、彝、朝鲜、维吾尔文等）的超大型中文编码字符集强制性标准，其中收入汉字70244，增加的这些内容是推荐性的；
*UTF（Unicode Transformation Format）：因国际组织发布的unicode仅仅为一种编码方式，不能很快普及，且部分与ASCII字符集不兼容等等一系列问题，所以产生了其unicode的实现方式，称为UTF，它是中间媒介，将unicode的编码规则与计算机的实际编码对应起来的规则，分为UTF-7，UTF-8，UTF-16，UTF-32，其中常用的为UTF-8和UTF-16；
  • UTF-7:7位 Unicode转换格式，是一种可变长度的字符编码，用于使用ASCII字符流来表示Unicode文本。它最初旨在提供一种编码Unicode文本的手段，用于互联网电子邮件消息，其效率高于UTF-8与可引用打印的组合；
  • UTF-8:能够编码所有可能的Unicode 码点的字符编码，灵活的变长编码，对于 ASCII 使用一个字节编码，其他本地化语言文字用多个字节编码，最长可以到 6 个字节编码一个字符。对于汉字，通常是 3 个字节表示一个汉字。这是 Unix/Linux 系统默认的字符编码。
  • UTF - 16 ：一般都是两字节表示一个字符，对于超出两字节的国际码字符，使用一对两字节来表示。在存储时，按两个字节的排布顺序，分为大端字节序和小端字节序；

>

  • LitterEndianUnicode （小端字节序）: 低字节在前，高字节在后
  • BigEndianUnicode （大端字节序）: 高字节在前，低字节在后
  • BOM（Byte Order Mark），字节顺序标记,为了识别 Unicode 文件，Microsoft 建议所有的 Unicode 文件应该以 ZERO WIDTH NOBREAK SPACE（U+FEFF）字符开头。这作为一个“特征符”或“字节顺序标记（byte-order mark，BOM）”来识别文件中使用的编码和字节顺序，Linux/UNIX 并没有使用 BOM，因为它会破坏现有的 ASCII 文件的语法约定，window不同编辑器对BOM处理也不相同；
  • 两者存储顺序不同，如"A"的Unicode编码为6500，而BigEndianUnicode编码为0065；
  • UTF - 32: UCS-4，因为用四个字节表示一个字符，所以不需要考虑扩展了。这种编码方式简单，但也特别浪费空间，所以应用很少。在存储时也分为 UTF-32BE 和 UTF-32LE，因为用得少，所以不用太关心这种编码格式。
*ISO8859：一系列的标准，以同样的码位对应不同字符集。
* Shift_JIS ：日本电脑系统常用的编码表。
*Big5：大五码，是通行于港澳台地区的一个繁体字编码方案。
* KOI8-U：KOI-8系列的斯拉夫文字8位元编码，供乌克兰语使用；
* KOI8-R：KOI-8系列的斯拉夫文字8位元编码，供俄语及保加利亚语使用；
